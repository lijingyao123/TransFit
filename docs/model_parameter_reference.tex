\section*{Legacy Forward-Call Short Forms}

For backward compatibility, forward-model helpers still accept shorter \texttt{params} dicts for some models:

\begin{itemize}
  \item \texttt{nickel}: omitting \texttt{E\_Th\_in} and \texttt{R\_0} maps them to \texttt{0} and \texttt{10 R\_sun};
  \item \texttt{magnetar}: omitting \texttt{E\_Th\_in} and \texttt{R\_0} maps them to \texttt{0} and \texttt{1 R\_sun};
  \item bolometric forward helpers may omit \texttt{T\_floor}; TransFit then uses an internal numerical floor.
\end{itemize}

Fitting APIs use the canonical parameter sets above, except that \texttt{fit\_bol()} keeps \texttt{T\_floor} internal and does not fit it.

\section*{Physical Parameter Constraints}

TransFit validates public forward calls and fit-time fixed/sampled values against these model-independent constraints before using them in the solver:

\begin{itemize}
  \item \texttt{M\_ej}, \texttt{v\_ej}, \texttt{R\_0}, \texttt{kappa}, \texttt{kappa\_gamma}, \texttt{T\_floor}, \texttt{P\_ms}, and \texttt{B14} must be finite and strictly positive;
  \item \texttt{E\_Th\_in} and \texttt{M\_Ni} must be finite and non-negative;
  \item \texttt{x\_Ni} must be finite and in \texttt{[0, 1]};
  \item if both are present, \texttt{M\_Ni <= M\_ej};
  \item \texttt{t\_shift} must be finite.
\end{itemize}

The solver output is also checked before interpolation or photometry. Non-positive or non-finite \texttt{Lbol}, \texttt{Teff}, or \texttt{Rph} values are rejected instead of being silently clipped into log-space interpolation.

\section*{\texttt{t\_shift} Convention}

In fitting mode, the likelihood is evaluated as
\[
\mathrm{model}(t_{\mathrm{obs}} + t_{\mathrm{shift}}).
\]

Therefore:
\begin{itemize}
  \item positive \texttt{t\_shift} shifts the model curve to earlier observed times;
  \item if you do not want to fit \texttt{t\_shift}, fix it with \texttt{fixed = \{"t\_shift": 0.0\}}.
\end{itemize}
